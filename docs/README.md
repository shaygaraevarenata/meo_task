## Training Planner — План тренировок

**Автор:** Шайгараева Рената Ильгизовна  
**Дата создания:** Май 2026

## Описание

**Training Planner** — это GUI‑приложение для планирования тренировок с фильтрацией, сохранением данных в формате JSON и поддержкой Git.

Программа позволяет:
* добавлять тренировки с указанием даты, типа и длительности;
* фильтровать тренировки по типу и дате;
* сохранять данные в JSON‑файл;
* загружать данные из JSON‑файла при запуске;
* проверять корректность ввода данных (формат даты, положительная длительность).

## Требования

* Python 3.6 или выше;
* стандартная библиотека `tkinter` (встроена в Python);
* библиотека `json` (встроена в Python).

## Инструкция по установке и запуску

### Шаг 1. Проверка наличия Python

1. Откройте командную строку (Windows: `Win + R` → введите `cmd` → Enter) или терминал (macOS/Linux).
2. Проверьте версию Python:
```bash
python --version
```
или
```bash
python3 --version
```
3. Если Python не установлен или версия ниже 3.6, скачайте и установите Python с официального сайта: https://python.org.

**Важно:** при установке отметьте галочку **«Add Python to PATH»**.

### Шаг 2. Клонирование репозитория

1. Откройте терминал или командную строку.
2. Перейдите в нужную директорию (например, в папку `Projects`):
```bash
cd ~/Projects
```
(для Windows: `cd C:\Users\ВашеИмя\Projects`)
3. Клонируйте репозиторий:
```bash
git clone <ссылка_на_репозиторий>
```
4. Перейдите в папку проекта:
```bash
cd <название_папки_проекта>
```

### Шаг 3. Проверка структуры проекта

Убедитесь, что в папке проекта присутствуют следующие файлы:
* `training_app.py` — главный файл приложения;
* `data_handler.py` — модуль для работы с данными;
* `README.md` — текущая инструкция;
* `.gitignore` — настройки игнорирования файлов;
* (опционально) `requirements.txt` — файл зависимостей.

### Шаг 4. Запуск приложения

1. Убедитесь, что вы находитесь в папке проекта.
2. Запустите приложение командой:
```bash
python training_app.py
```
или (для Linux/macOS):
```bash
python3 training_app.py
```

### Шаг 5. Первое использование

После запуска откроется окно приложения. Для начала работы:

1. **Добавление тренировки:**
   * введите дату в формате `YYYY-MM-DD` (например, `2026-05-15`);
   * выберите тип тренировки из выпадающего списка;
   * укажите длительность в часах (положительное число);
   * нажмите кнопку **«Добавить тренировку»**.

2. **Фильтрация данных:**
   * выберите тип тренировки в фильтре;
   * при необходимости укажите диапазон дат («Дата с» и «по»);
   * нажмите **«Применить фильтр»**;
   * для сброса фильтров нажмите **«Сбросить фильтр»**.

3. **Автоматическое сохранение:**
   * все добавленные тренировки автоматически сохраняются в файл `data/trainings.json`;
   * данные загружаются при каждом запуске приложения.

## Возможные проблемы и решения

| Проблема | Решение |
|--------|---------|
| `python: command not found` | Используйте `python3` вместо `python`. |
| Ошибка импорта tkinter | Убедитесь, что Python установлен полностью (tkinter входит в стандартную библиотеку). |
| Не создаётся папка `data` | Приложение создаст её автоматически при первом сохранении данных. |
| Данные не сохраняются | Проверьте права доступа к папке проекта; убедитесь, что файл `data/trainings.json` не защищён от записи. |

## Дополнительные рекомендации

* **Резервное копирование данных:** файл `data/trainings.json` содержит все данные о тренировках. Рекомендуется периодически делать его резервные копии.
* **Работа с Git:** проект поддерживает Git. После внесения изменений вы можете зафиксировать их:
```bash
git add .
git commit -m "Добавлены новые тренировки"
git push
```
* **Обновление приложения:** чтобы получить последние версии кода, выполните:
```bash
git pull
```

## Структура данных JSON

Данные сохраняются в файле `data/trainings.json` в следующем формате:
```json
[
  {
    "id": 1,
    "date": "2026-05-10",
    "type": "Бег",
    "duration": 1.5
  },
  {
    "id": 2,
    "date": "2026-05-12",
    "type": "Плавание",
    "duration": 0.75
  }
]
```
